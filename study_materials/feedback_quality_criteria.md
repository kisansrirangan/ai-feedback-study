The generated feedback should be assessed against these feedback quality criteria:

1. Accuracy
   - Correctly identifies real strengths and weaknesses in the student's code.
   - Does not invent errors when the code is correct.

2. Specificity
   - Refers to concrete aspects of the submitted code.
   - Avoids overly generic feedback.

3. Actionability
   - Helps the student understand what to improve next.
   - Gives clear and practical guidance.

4. Understandability
   - Uses plain language suitable for an introductory programming student.
   - Avoids unnecessary technical complexity.

5. Pedagogical usefulness
   - Supports learning rather than only judging the answer.
   - Prioritises the most important issue first.
